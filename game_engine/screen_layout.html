游戏逻辑屏幕布局¶

The design, construction, debugging and running of a game utilizes a wide range of Blender functions. To help with the process, Blender incorporates a suggested screen layout for setting up BGE games. This includes many already-familiar panels but also a new 逻辑编辑器 panel (4) concerned solely with the BGE.

The diagram below shows this default Game 逻辑 screen layout, together with the appropriate options for game setup/debug/running (these should be set up in the order shown).

游戏逻辑屏幕布局.

Game 逻辑 Menu.

1) Game 逻辑

Selected from the list of screen layouts for various applications. This includes many already-familiar panels Information, 3D视图, Properties but also a new 逻辑编辑器 panel concerned solely with the BGE.

Selected from the render engine menu. This specifies that all output will be output by the real-time Blender 游戏引擎 renderer. It also opens various other menu options such as the Game options (see below) and a range of Properties for the BGE renderer properties (see below)

4) 逻辑编辑器 panel

The 逻辑编辑器 is where the logic, properties and states are set up to control the behaviour of the objects in the game. (The 逻辑编辑器 panel can also be displayed by selecting 逻辑编辑器 in the Display Editor menu, by pressing Shift-F2, or by pressing Ctrl-Right).

5) Properties

Tip

Two Meanings for the Same Word

Note that the name “Property” has two different uses in Blender terminology - firstly in the wider use of the Property Display Panel as described here, and secondly as the term used for specific 游戏引擎 logic variables which are also called “properties”.

The Property panel of the screen is selected as usual from the main Information menu. However, note that several sections of the Property panel are changed when the render engine (2) is changed from Blender 渲染 to Blender Game.
